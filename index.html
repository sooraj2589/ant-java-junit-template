Ant, Java, JUnit, Cobertura and Checkstyle build template

Writing build files can be tedious. Especially if it is something you are not familiar with, or just if you want to start writing code as soon as possible, and you do not have time for other things.

The purpose of this template is to provide a complete build process for Java projects, using Apache Ant as build system. Copy it, change the project name in the build file, and that's all.

If you are thinking to use a Continuous Integration system for your Java project, this is what you have been looking for. And, if you were not thinking in Continuous Integration, now you have the chance to start it, having almost all the work done.

This template includes the following software engineering practices automated in the build process:

  • Unit testing, with JUnit.
  • Coverage analysis, with Cobertura.
  • Code style validation, with Checkstyle.
